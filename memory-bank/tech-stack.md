## 逐层选型

### 1. 前端框架：uni-app (Vue 3)

| 项 | 选型 | 理由 |
|----|------|------|
| 框架 | **uni-app 3.x** (Vue 3 Composition API) | 用 Vue 语法开发，编译为微信小程序原生代码；后续可一键出 H5/App |
| UI 组件 | **uni-ui**（uni-app 官方组件库） | 替代 WeUI（原生小程序组件库在 uni-app 中不可用），组件齐全，与 uni-app 深度集成 |
| 状态管理 | **Pinia** | Vue 3 推荐的状态管理库，替代 Vuex；轻量、类型友好、支持 composition API |
| 构建工具 | **HBuilderX** 或 **Vue CLI + uni-app 插件** | HBuilderX 对小程序编译支持最好；CLI 方式适合团队协作和 CI |

**为什么不用原生小程序 + WeUI：** 原生小程序使用 WXML/WXSS/JS 模板语法，无法用 Vue 组件体系。uni-app 是 Vue → 小程序的标准桥梁。

### 2. 后端/运行时：微信云开发·云函数 (Node.js)

| 项 | 选型 | 理由 |
|----|------|------|
| 运行时 | **Node.js 18** | 微信云开发支持的最新 LTS 版本 |
| 函数框架 | **wx-server-sdk** | 微信官方 SDK，封装了云数据库、云存储、云调用的操作 |
| 函数拆分 | 按领域垂直拆分（见下文） | 每个云函数独立部署，互不干扰 |

**为什么没有 SpringBoot：** 微信云数据库的 HTTP API 与 SpringBoot 生态（JPA/MyBatis）不匹配，通过 HTTP API + access_token 的方式访问云数据库复杂且受限。云函数直连云数据库是最自然的路径。

### 3. 数据存储：微信云数据库 (NoSQL)

| 项 | 选型 | 理由 |
|----|------|------|
| 数据库 | **微信云数据库**（文档型 NoSQL） | 免运维，与云函数零延迟内网互通，无需管理连接池 |
| 数据模型 | 集合（Collection）= 表，文档（Document）= 行 | 与 MongoDB 类似，JSON 结构存储 |

**云数据库 vs MySQL 对比：**

| 维度 | 云数据库 | MySQL |
|------|---------|-------|
| 运维 | 零运维，自动扩缩 | 需自建/云服务器部署 |
| 查询能力 | 简单查询强，聚合弱 | 完整 SQL，JOIN/分组/子查询 |
| 数据结构 | JSON 文档，天然适合题目/知识点 | 需建表建字段 |
| MVP 适合度 | ✅ 极高（零运维、快速迭代） | ❌ 过度设计 |

**本项目数据特点：** 题目和知识点是典型的"读多写少"JSON 文档，云数据库完全匹配。

### 4. 静态资源：微信云存储

| 项 | 选型 | 理由 |
|----|------|------|
| 图标 | **云存储** 或 **iconfont 字体图标** | iconfont 体积小，不占存储；复杂图标可上云存储 |
| 题库封面图 | **云存储**（CDN 加速） | 微信自带 CDN，加载快 |
| 空状态插画 | **云存储** | 集中管理，方便替换 |

### 5. 认证与权限：免登录 + 匿名访问

| 项 | 选型 | 理由 |
|----|------|------|
| 用户标识 | **wx-openid**（匿名获取） | 云函数中通过 `cloud.getWXContext().OPENID` 自动获取，无需用户授权 |
| 登录 | **无** | MVP 不做登录，openid 仅用于区分不同用户数据（错题/收藏隔离） |
| 数据权限 | 云数据库 `_openid` 字段 | 每条用户数据标记 openid，读时按 openid 过滤 |

**关键变化 vs 原 PRD：** 原 PRD 说"数据存本地缓存"。引入云数据库后，错题和收藏数据改为**云端存储**（按 openid 隔离），好处是卸载重装不丢数据。刷题进度仍可保留本地缓存以减少请求次数，但核心数据在云端。

### 6. 部署与发布

| 项 | 选型 | 理由 |
|----|------|------|
| 小程序端 | **微信开发者工具** 上传 → 微信审核 | uni-app 编译产物为标准小程序代码 |
| 云函数 | **微信开发者工具** 一键部署 | 每个云函数独立上传，在线编辑/调试 |
| 版本管理 | Git（前端 + 云函数代码在同一仓库） | 见 `Q6 仓库定位` 决策 |

### 7. 代码高亮方案

| 项 | 选型 | 理由 |
|----|------|------|
| 高亮核心 | **highlight.js**（剪裁版） | 只保留 Excel/ Python/ Shell 三种语言，打包体积 ~20KB |
| 渲染方式 | uni-app `<rich-text>` 组件 | highlight.js 预编译 → 高亮 HTML 字符串 → rich-text 渲染 |

**为什么不是 towxml：** towxml 面向原生小程序 WXML，uni-app 有自己的 `rich-text` 组件，直接使用更简洁。

### 8. 测试

| 项 | 选型 | 理由 |
|----|------|------|
| 前端单元测试 | **Vitest** | 与 Vite 生态一致，对 Vue 3 支持好 |
| 云函数测试 | **微信云开发本地调试** + **Jest** | 微信开发者工具支持本地模拟云函数调用 |
| UI 验收 | **真机测试**（iOS + Android） | 微信开发者工具模拟器 + 真机扫码 |

## 云函数规划

| 函数名 | 触发方式 | 功能 |
|--------|---------|------|
| `getBanks` | HTTP 调用 | 返回题库注册列表（名称、题量、封面图） |
| `getQuestions` | HTTP 调用 | 按 bankId + 分页获取题目列表 |
| `getKnowledgeCategories` | HTTP 调用 | 返回速查分类列表 |
| `getKnowledgeItems` | HTTP 调用 | 按 categoryId 获取知识点列表 |
| `getKnowledgeDetail` | HTTP 调用 | 按 id 获取知识点详情 |
| `search` | HTTP 调用 | 全局模糊搜索（搜索题目 + 知识点） |
| `syncWrongQuestions` | HTTP 调用 | 同步错题数据（上传/下载） |
| `syncFavorites` | HTTP 调用 | 同步收藏数据（上传/下载） |

## 与设计文档的关键差异

引入云开发后，与原始 PRD/设计文档有以下调整：

| 原方案 | 新方案 | 影响 |
|--------|--------|------|
| 题库/手册 JSON 打包在小程序内 | 题库/手册存云数据库，云函数下发 | 内容更新无需发版 |
| 错题/收藏仅本地缓存 | 云端存储 + 本地缓存双写 | 卸载不丢数据 |
| 搜索为本地检索 | 搜索走云函数（云数据库全文检索） | 搜索能力更强，但需网络 |
| 不申请用户权限 | 云函数自动获取 openid | 静默获取，用户无感 |
