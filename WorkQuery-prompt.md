# 职场刷题速查小程序 开发设计文档（Vue3 + SpringBoot 版）
## 一、项目概述
### 1.1 产品定位
面向职场人的轻量化微信小程序，聚焦**碎片化刷题、工作速查、面试突击**三大核心场景，主打「短平快、无干扰、随手用」，精准适配职场人「摸鱼练题、工作查语法/参数、面试前临时抱佛脚」的真实需求。

项目采用**前后端分离架构**：前端基于 Vue3 生态的 uni-app 框架开发，编译为微信小程序；后端基于 Java SpringBoot 构建 RESTful 服务，提供稳定的数据支撑与业务能力。

### 1.2 目标用户与核心场景
| 用户群体 | 核心使用场景 |
| :--- | :--- |
| 互联网从业者（开发、测试、产品、运营等） | 通勤/工位碎片化时间刷题；工作中快速查询语法、命令、规范 |
| 求职/跳槽人群 | 面试前1-2周集中刷高频题；临面试前快速过核心考点 |
| 泛职场技能学习者 | 利用碎片时间积累专业知识，随时查阅工具用法、行业规范 |

### 1.3 核心设计原则
1. **极致轻量化**：小程序首屏加载 < 1s，核心功能3步内触达，无冗余弹窗与广告
2. **场景化分区**：三大核心功能入口清晰，用户可快速定位需求
3. **隐蔽友好**：界面简洁克制，低饱和度配色，适配职场「低调使用」场景
4. **离线可用**：核心题库与速查内容支持本地缓存，弱网/无网环境可正常使用
5. **接口高可用**：后端接口响应耗时 < 200ms，支持高并发刷题、搜索场景

### 1.4 整体架构图
```
[微信小程序端 (Vue3 + uni-app)]
        ↓ HTTPS
[SpringBoot 后端服务]
        ↓
[MySQL 主数据库] ←→ [Redis 缓存层]
```

## 二、技术栈选型
### 2.1 前端技术栈（Vue3 生态）
基于 Vue3 语法开发，通过 uni-app 跨端框架编译为微信小程序，兼顾开发效率与小程序生态兼容性。
| 技术/工具 | 选型说明 |
| :--- | :--- |
| 核心框架 | Vue 3.x + Composition API + uni-app 3.x |
| 状态管理 | Pinia（替代 Vuex，更轻量，原生支持 Vue3） |
| UI 组件库 | uView Plus（适配 Vue3 + uni-app，组件丰富，按需引入） |
| 网络请求 | 基于 uni.request 二次封装（拦截器、Token 自动携带、异常统一处理） |
| 代码高亮 | Prism.js 轻量适配版，支持多编程语言代码渲染 |
| 本地存储 | uni.storage + 持久化插件，存储刷题进度、缓存题库、用户Token |
| 工程化 | Vite 构建、ESLint 代码规范、小程序分包加载优化 |

### 2.2 后端技术栈（Java SpringBoot 生态）
采用成熟稳定的 SpringBoot 技术体系，兼顾开发效率与系统扩展性。
| 技术/工具 | 选型说明 |
| :--- | :--- |
| 核心框架 | Spring Boot 3.x（JDK 17） |
| ORM 框架 | MyBatis-Plus 3.5.x（简化 CRUD，代码生成器提升开发效率） |
| 数据库 | MySQL 8.0（关系型数据库，存储题库、用户、答题记录等核心数据） |
| 缓存中间件 | Redis 7.x（缓存热门题库、搜索热词、用户Token，提升接口响应速度） |
| 鉴权方案 | JWT + Spring Security（无状态鉴权，适配小程序多端访问） |
| 接口文档 | Knife4j 4.x（基于 Swagger，自动生成在线接口文档，方便前后端联调） |
| 工具类库 | Hutool、Lombok（简化代码，提升开发效率） |
| 参数校验 | Spring Validation（统一入参校验，异常全局拦截） |
| 定时任务 | Spring Task（每日生成每日一练、统计用户数据、更新缓存） |
| 数据库连接池 | HikariCP（SpringBoot 默认，高性能轻量） |

## 三、核心功能模块设计
### 3.1 用户中心模块
#### 前端功能
- 微信一键授权登录，自动同步微信头像、昵称，无需手动注册
- 个人数据面板：累计刷题数、正确率、连续打卡天数、速查次数
- 我的收藏：题目收藏、知识点收藏统一分类管理
- 设置中心：字体大小、暗色模式、消息通知开关、清除缓存、意见反馈

#### 后端能力
- 微信登录接口：校验微信 code，获取 openid，自动注册/更新用户信息，返回 JWT Token
- 用户信息管理：查询/更新用户基础信息、学习数据统计
- 收藏管理：新增/取消收藏、分页查询收藏列表
- 反馈提交：接收用户题库纠错、功能建议，存入反馈表

### 3.2 碎片化刷题模块
主打「单次1-5分钟、随时中断随时续」，适配通勤、午休、工位摸鱼等短时间场景。

#### 前端功能
- **题库分类体系**：按岗位→知识点二级分类展示，支持难度筛选
- **多种刷题模式**：
  - 随机刷题：自定义题量（5/10/15题），从选定分类随机出题
  - 顺序刷题：按知识点顺序逐题练习，自动记录刷题进度
  - 每日一练：每日推送5道精选高频题，完成后打卡
  - 背题模式：直接展示题目+答案+解析，适合快速突击记忆
- **答题交互**：选完即出答案，点击展开解析；上下滑动切换题目；右滑自动保存进度
- **错题与收藏**：答错自动收录错题本；支持手动收藏题目；错题批量重做，答对可移除
- **数据统计**：正确率趋势、各知识点掌握度雷达图、打卡日历

#### 后端能力
- 题库管理：题目增删改查、分类管理、难度/频率标签配置
- 抽题接口：根据分类、难度、题量参数，随机/顺序返回题目列表
- 答题记录：提交用户答案，自动判分，写入答题记录表，同步更新错题本
- 错题本接口：分页查询错题、错题重做、移除错题
- 统计接口：计算用户累计刷题数、正确率、知识点掌握度、打卡数据

### 3.3 工作速查模块
主打「秒搜、精准、无广告」，替代工作中翻文档、百度搜索的低效操作。

#### 前端功能
- **速查知识库**：分类展示编程语言语法、SQL/Linux/Git 命令、CSS属性、职场规范等
- **全局搜索**：首页顶部常驻搜索框，支持关键词模糊搜索、拼音联想、搜索历史
- **知识点详情**：卡片式展示语法、参数、示例代码，代码块支持一键复制、语法高亮
- **常用与历史**：自定义「我的常用」知识点，首页直达；自动记录最近查阅记录
- **离线缓存**：支持下载全部分类内容，无网络环境可正常查阅

#### 后端能力
- 知识库管理：知识点增删改查、分类管理、标签配置、热度统计
- 全文搜索：基于 MySQL 全文索引 / Elasticsearch（可选）实现关键词模糊搜索，按匹配度+热度排序
- 搜索热词：记录搜索频次，热门关键词自动置顶，搜索联想推荐
- 离线同步：提供全量/增量知识点拉取接口，支持前端本地缓存更新

### 3.4 面试突击模块
主打「短期高效、直击考点」，适配面试前集中备考、临考抱佛脚场景。

#### 前端功能
- **高频题合集**：按岗位/公司分类的TOP高频题集，标注出现频率、难度
- **卡片背诵模式**：正面题目、背面答案+答题思路，点击翻转，左右滑动切换，标记「已掌握/未掌握」
- **模拟面试**：随机抽取10/20道题组成模拟卷，计时作答，答完生成分数与薄弱点分析
- **面试倒计时**：设置面试日期，生成每日刷题计划，考前推送「必看清单」

#### 后端能力
- 题集管理：面试题集创建、题目关联、频率标签配置
- 学习进度：记录用户背诵进度、已掌握/未掌握题目，支持进度续接
- 模拟试卷：根据规则随机生成试卷，提交后判分并生成能力分析
- 面试计划：根据面试日期与题量，自动生成每日学习任务

- 知识点表：`title`、`tags` 建全文索引，优化搜索性能

## 五、核心接口设计规范
### 5.1 统一接口返回格式
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```
- `code`：状态码，200成功，401未登录，400参数错误，500服务器异常
- `message`：提示信息
- `data`：响应数据

### 5.2 鉴权规则
- 登录、公开题库、公开速查内容等接口无需鉴权
- 答题记录、收藏、个人数据、学习进度等接口需在请求头携带 `Authorization: Bearer {token}`
- Token 有效期 7 天，前端自动续期，过期返回 401 触发重新登录

### 5.3 核心接口示例
| 接口路径 | 请求方式 | 接口说明 | 鉴权 |
| :--- | :--- | :--- | :--- |
| /api/auth/wx-login | POST | 微信授权登录，获取Token | 否 |
| /api/question/categories | GET | 获取题库分类列表 | 否 |
| /api/question/random | GET | 随机抽题（分类、题量、难度参数） | 是 |
| /api/question/submit | POST | 提交答题答案，自动判分 | 是 |
| /api/question/wrong | GET | 分页获取错题本列表 | 是 |
| /api/knowledge/search | GET | 关键词搜索知识点 | 否 |
| /api/knowledge/detail/{id} | GET | 获取知识点详情 | 否 |
| /api/interview/sets | GET | 获取面试题集列表 | 否 |
| /api/user/statistics | GET | 获取用户学习统计数据 | 是 |
| /api/favorite/add | POST | 添加收藏 | 是 |

## 八、开发与部署方案
### 8.1 开发环境要求
- 前端：Node.js 16+、HBuilderX / VS Code、微信开发者工具
- 后端：JDK 17、Maven 3.8+、MySQL 8.0、Redis 7.x

### 8.2 前后端联调
- 后端启动后，通过 Knife4j 文档地址查看接口详情：`http://localhost:8080/doc.html`
- 前端配置开发环境代理，解决跨域问题；真机调试时后端需配置可访问的内网/公网地址
- 接口联调遵循「先定义接口文档，再并行开发，最后联调」的流程

### 8.3 生产部署
- **后端**：打包为 Jar 包，部署到云服务器，配合 Nginx 反向代理；配置 MySQL 主从、Redis 持久化保障数据安全
- **前端**：uni-app 编译为微信小程序代码包，通过微信开发者工具上传审核发布
- **域名与备案**：后端接口域名需完成 ICP 备案，配置 HTTPS 证书，符合微信小程序域名要求

## 九、性能与体验优化
### 9.1 前端优化
- 启用小程序分包加载，主包控制在 2MB 以内，核心功能放主包，次要功能放子包
- 题库、知识点数据做本地缓存，二次访问无网络请求
- 图片懒加载，非首屏图片延迟渲染；统一压缩静态资源
- 长列表使用虚拟滚动，避免大量 DOM 渲染卡顿

### 9.2 后端优化
- 热门题库、分类列表、高频搜索结果存入 Redis，减少数据库压力
- 答题记录、统计数据采用异步写入 + 定时聚合，避免接口耗时过长
- 数据库慢查询优化，关键查询建立合适索引
- 接口限流防刷，防止恶意请求冲击服务
